# January Corporation 정책 및 규칙

## 프로젝트 정책

### 1. 코드 구조 원칙

#### HTML 규칙
- 모든 HTML 페이지는 상대 경로 사용 (`./file.html`)
- 메뉴 항목명 통일: "January Corporation" → "CONTACT"
- 인라인 스타일 금지 - 외부 CSS 파일 사용
- jQuery 3.x 최신 버전 유지, migrate 제거

#### CSS 규칙
- 페이지별 커스텀 CSS 분리 (`css/*-custom.css`)
- 색상 팔레트 준수:
  - 주요색: #667eea (파란 보라), #764ba2 (보라)
  - 배경: #000000 (헤더), #f8f9ff (섹션), #ffffff (콘텐츠)
  - 텍스트: #000000 (제목), #555555 (본문), #999999 (보조)
- 비표준 CSS 선택자 금지 (`:contains()` 등)
- 반응형 디자인 필수:
  - 모바일: 768px 이하
  - 태블릿: 1024px 이하
  - 데스크톱: 1200px 기준

#### 파일명 규칙
- HTML: `{페이지명}.html` (소문자)
- CSS: `{페이지명}-custom.css`
- 이미지/리소스: 한글 허용, 확장자는 소문자

### 2. 파일 정리 규칙

- `.다운로드` 파일 정기적 삭제
- 불필요한 jQuery Migrate 파일 제거
- 사용하지 않는 CSS/JS 파일 정리

### 3. 메뉴 네비게이션 규칙

현재 메뉴 구조:
```
- NEWSROOM (뉴스룸)
- Work (포트폴리오)
- CONTACT (연락처)
- BUSINESS (사업소개)
- COMPANY (회사소개)
```

변경 금지 - 모든 HTML에서 동일하게 유지

### 4. 커밋 메시지 컨벤션

포맷:
```
카테고리: 변경 내용

상세 설명 (필요시)

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

카테고리:
- `최적화`: 성능, 파일 크기, 로딩 시간
- `기능`: 새로운 기능 추가
- `수정`: 버그 또는 문제 해결
- `스타일`: UI/디자인 개선
- `정리`: 코드 리팩토링, 구조 개선
- `문서`: README, 주석 etc

### 5. 품질 기준

#### 성능
- 페이지별 순 콘텐츠 크기: 50KB 이하 (이상적)
- 리소스 최소화 (불필요한 라이브러리 제거)
- 이미지 최적화 필수

#### 접근성
- 모든 이미지에 alt 텍스트
- 시맨틱 HTML 사용
- 색상 대비 충분히

#### SEO
- 메타 태그 유지
- Naver 사이트 인증 코드 유지
- 구조화된 데이터 (Schema.org)
